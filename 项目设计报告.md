# LED 风扇双通道智能控制器 — 项目设计报告

## Context（背景）

本项目是一个基于 Renesas RA6E2 (R7FA6E2BB) MCU 的 **双通道 LED 风扇供电/监控/保护系统**。系统接收 DC 电源输入，经过两级降压后为 MCU 和外设供电，同时通过两路独立的继电器通道为外部 LED 风扇提供可控电源。每条通道均配备 INA226 高精度电流/电压监测芯片，实现对负载的实时监控与故障保护。系统还通过 UART 驱动串口屏实现人机交互，用户可通过屏幕上的滑动条实时调节风扇转速。

硬件包含：DC-DC 降压电源、LDO 线性稳压、INA226 电流/电压监测、继电器功率通断、PWM 风扇调速、串口屏人机交互、红/绿双色 LED 状态指示。

软件基于 Renesas FSP 裸机架构，实现完整的"采集 → 判断 → 保护 → 控制 → 显示"闭环。

---

## 一、硬件设计要点

### 1.1 电源设计（两级降压）

#### 第一级：DC-DC Buck — TPS62933FDRLR (U46)

TPS62933 是 TI 推出的 3.8V~30V 输入、3A 输出同步降压转换器，内置高低侧 MOSFET，开关频率 2.2MHz。本项目用它将外部 DC 输入（设计裕量支持 12V~24V 宽范围）高效降压至 5V。

- **输入端**：DC 电源插座 DC-005-A200 (额定 3A/30V，内径 2mm / 外径 6.4mm)
- **输入滤波**：
  - C170/C171 (2×10uF 铝电解, SMD 3×5.4mm) — 大容量储能，吸收输入纹波
  - C172/C173 (2×100nF 陶瓷, 1206 封装) — 高频去耦，抑制开关噪声回灌
  - 电解 + 陶瓷并联组合兼顾低频纹波和高频噪声
- **输出**：5V / 3A
- **开关频率**：2.2MHz（高频优势：电感/电容体积小，EMI 易于滤波）
- **反馈网络**：R98 (53.6K) + R99 (10.2K) → Vout = 0.6V × (1 + 53.6K/10.2K) ≈ 3.76V（实为 5V 配置，内部参考电压 0.6V）
- **使能脚 (EN)**：由 R97 (100K) 上拉至 VIN，实现上电自动启动，无需 MCU 控制
- **软启动**：C165 (33nF) 控制启动斜率，限制浪涌电流，保护后级电路
- **自举电容**：C166 (100nF) 为内部高侧 N-MOSFET 提供浮动栅极驱动电压
- **RT 引脚**：R96 (47K) 连接至 RT 脚，设定开关频率（部分型号支持外部电阻调频）
- **输出滤波**：
  - L1 (SRG0420-2R2-M, 2.2uH 一体成型电感, 4A 额定/5A 饱和, DCR 49mΩ) — 低 DCR 减少铜损，一体成型结构抗电磁辐射
  - C167 (10uF MLCC) + C169 (47uF MLCC) — 主输出滤波，MLCC 低 ESR 降低输出纹波
  - C168 (100nF) — 高频旁路

#### 第二级：LDO — TPS7A8101DRBR (U30)

TPS7A8101 是 TI 的低噪声高 PSRR LDO，输出电流 1A，压差仅 500mV@1A。选此芯片为 3.3V 数字/模拟混合电路供电，核心目的是提供干净的电源轨给 INA226 和 MCU。

- **输入**：5V（来自第一级输出），使能脚 (EN) 直连 5V，上电即工作
- **输出**：3.3V / 1A
- **反馈网络**：R65 (30.9K) + R66 (10K) → Vout = 0.8V × (1 + 30.9K/10K) ≈ 3.27V，标称 3.3V
- **噪声旁路电容**：C24 (10nF) 接 NR 脚（Noise Reduction），进一步滤除内部参考源噪声，将输出噪声压至 23.5uVrms
- **输出滤波**：C22/C23 (2×10uF MLCC) 确保 LDO 环路稳定性
- **关键指标**：
  - PSRR = 80dB@1kHz — 开关纹波经此级衰减 10000 倍，对后级 INA226 测量精度至关重要
  - 输出噪声 23.5uVrms — 低噪声设计，避免电源噪声叠加到模拟测量信号上

### 1.2 电流/电压监测 — INA226AIDGSR × 2 (U49, U50)

INA226 是 TI 的高精度双向电流/功率监测器，16 位分辨率，I2C 数字接口，支持高侧和低侧电流检测。本项目使用两片 INA226 分别监测两路风扇通道。

#### 地址配置

| 芯片 | A1 引脚 | A0 引脚 | 7-bit 地址 | 监测通道 |
|------|---------|---------|-----------|---------|
| U49  | GND     | VCC(3V3)| 0x41      | 通道 1  |
| U50  | GND     | GND     | 0x40      | 通道 2  |

- A0/A1 引脚通过上拉/下拉电阻配置，支持 16 种地址组合

#### 采样电阻设计

- **阻值**：R104/R105 = 200mΩ (0.2Ω)，1206 封装
- **功率计算**：最大电流 90mA（保护阈值），P = I²R = (0.09)² × 0.2 = 1.62mW，远低于 1206 封装 250mW 额定功率，裕量充足
- **压降计算**：满量程压降 = 90mA × 200mΩ = 18mV，INA226 分流电压 LSB = 2.5uV，理论分辨率 = 18mV / 2.5uV = 7200 级，精度充足

#### PCB 开尔文 (Kelvin) 四线接法

电流采样电阻的 PCB 布局是本项目硬件设计的关键细节。INA226 的 VIN+ 和 VIN- 差分输入需要精确采集采样电阻两端的毫伏级压降，任何走线电阻引入的额外压降都会直接叠加为测量误差。因此必须采用**开尔文 (Kelvin) 四线接法**：

**原理说明：**

传统二线接法中，大电流通路与电压检测共用同一条走线。PCB 走线（铜箔）本身有电阻（典型值：1oz 铜厚 1oz 铜箔 1mm 宽走线约 50mΩ/cm），大电流流过时产生的 IR 压降会叠加到采样电阻两端的检测电压上，造成测量误差。

开尔文接法将**电流路径**和**电压检测路径**完全分离：

```
                    大电流路径（宽走线）
    VIN+ ──────┤                    ├────── V_RELAY
               │    R_sense         │
               │   (200mΩ)         │
    VIN- ──────┤                    ├────── GND_power
                    大电流路径（宽走线）

    INA226 电压检测路径（窄走线，就近采样）
    VIN+ 脚 ←── 直接从采样电阻焊盘内侧引出
    VIN- 脚 ←── 直接从采样电阻另一端焊盘内侧引出
```

**PCB 实施要点：**

1. **采样电阻焊盘设计**：R104/R105 (1206 封装) 的两端焊盘各引出两条独立走线：
   - **电流走线**：宽走线（≥1mm）连接至 VIN+ 电源平面和 V_RELAY/GND_power 节点，承载负载电流
   - **检测走线**：窄走线（0.25mm 即可）直接连接 INA226 的 VIN+ (Pin 10) 和 VIN- (Pin 9) 引脚，不经过任何大电流铜箔

2. **检测走线从焊盘内侧引出**：电压检测线必须从采样电阻焊盘的**内侧**（靠近电阻体一侧）引出，而非从电流走线的铜箔上分支，否则仍会引入走线电阻误差

3. **差分走线对称**：VIN+ 和 VIN- 的检测走线应等长、等宽、平行走线，且紧邻走线，使两根走线拾取的电磁干扰相同（共模干扰），INA226 的 140dB CMRR 可将其有效抑制

4. **检测走线远离大电流铜箔**：避免大电流走线的磁场耦合到检测走线中产生感应电压

5. **接地分离**：INA226 的 GND (Pin 7) 应通过独立走线连接到采样电阻的功率地（VIN- 端），而非共用大面积地铜，避免地线压降引入误差

**误差估算（不采用开尔文接法时）：**

若用 1cm 走线（1oz 铜, 0.5mm 宽, 约 100mΩ）替代开尔文接法，90mA 负载电流产生的额外压降 = 0.09A × 0.1Ω = 9mV，而采样电阻本身压降仅 18mV，误差高达 50%！开尔文接法将此误差降至可忽略水平。

#### I2C 总线设计

- **总线速率**：100kHz（Standard Mode），I2C 时钟配置：BRL=244, BRH=243, CKS=2，I2C 时钟源 50MHz
- **上拉电阻**：R100/R101 = 4.7K，上拉至 3.3V
  - 计算：t_rise ≈ 0.847 × R × C_bus，4.7K 对典型 100pF 总线电容 → t_rise ≈ 400ns，满足 Standard Mode ≤1000ns 要求
- **ALERT 中断**：R102/R103 (4.7K) 上拉，连接至 MCU GPIO（U48 Pin 5/6），预留硬件过流中断触发能力（当前软件未启用，但硬件已预留）
- **总线拓扑**：两片 INA226 共用同一 I2C 总线（SCL/SDA 并联），通过不同地址区分

#### INA226 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 分流电压 LSB | 2.5uV | 电流分辨率基础 |
| 总线电压 LSB | 1.25mV | 电压分辨率 |
| 共模电压范围 | 300mV ~ 40V | 支持宽范围供电电压 |
| CMRR | 140dB | 极强的共模噪声抑制 |
| 输入失调电压 | 2.5uV | 极低，对小电流测量精度影响小 |
| 静态电流 | 330uA | 低功耗 |

### 1.3 继电器功率通断控制

#### 继电器选型

- **型号**：HK19F-DC5V-SHG(Y) × 2 (K1, K2)，汇科品牌
- **触点形式**：双刀双掷 (DPDT, 2C)
- **触点容量**：1A@125VAC / 2A@30VDC
- **线圈参数**：5V 驱动，线圈电阻 45Ω → 线圈电流 = 5V / 45Ω ≈ 111mA，线圈功率 200mW
- **动作时间**：吸合 6ms，释放 4ms
- **接触电阻**：100mΩ

#### 驱动电路

```
MCU GPIO (RELAY1/2)
    │
    ├─ R106/R108 (1K) ──→ Q1/Q2 基极 (S8050 NPN)
    │
    └─ R107/R109 (10K) ──→ GND  (基极下拉，确保MCU复位期间三极管可靠关断)

Q1/Q2 集电极 ──→ 继电器线圈 (K1/K2 Pin 16) ──→ 5V
Q1/Q2 发射极 ──→ GND

D1/D2 (1N4148W) 反向并联于继电器线圈两端（阳极接集电极/线圈端，阴极接 5V）
```

- **基极限流**：R106/R108 (1K)，MCU GPIO 输出 3.3V，基极电流 = (3.3V - 0.7V) / 1K = 2.6mA
- **三极管增益**：S8050 hFE ≥ 120@50mA，集电极电流 111mA 时需要基极电流 ≥ 111/120 ≈ 0.93mA，2.6mA 基极电流保证三极管深度饱和（VCE(sat) ≈ 0.6V）
- **下拉电阻**：R107/R109 (10K)，MCU 上电/复位期间 GPIO 处于高阻态时，确保三极管基极为低电平，继电器不会误动作
- **续流二极管**：D1/D2 (1N4148W, 100V/150mA, Trr=4ns) — 继电器线圈断电瞬间产生反向电动势（可达数十伏），续流二极管将其钳位在约 0.7V，保护三极管免受击穿

#### 功率路径（信号流）

```
DC输入 (VIN) → INA226 采样电阻 R104/R105 → 继电器触点 (K1-8/K2-8 → K1-4/K2-4) → CN1/CN2 风扇接口
                                                     ↑
                                          INA226 VBUS 检测点（VIN- 之后）
```

- 继电器的常开触点串联在功率路径中，关断时物理切断负载电源，实现真正的电气隔离
- INA226 的 VIN+ 连接至 VIN（采样电阻前），VIN- 连接至 V_RELAY（采样电阻后、继电器前），监测的是**采样电阻上的压降**（即负载电流）；VBUS 连接至 V_RELAY，监测的是**继电器前的负载电压**

### 1.4 风扇接口 (CN1, CN2)

- **接插件**：KF2510-4AGW-GW (1×4P, 2.54mm 间距, 弯插, 额定 3A)
- **引脚定义**（从网表分析）：
  - Pin 1：GND
  - Pin 2：V_FAN（继电器输出的风扇电源）
  - Pin 3：未使用 / 信号线
  - Pin 4：PWM（来自 MCU GPT 定时器输出）
- **设计考量**：每路风扇采用"电源 + PWM"双线控制，电源通断由继电器控制，转速由 PWM 控制，两者独立，实现灵活的启停和调速

### 1.5 串口屏接口 (H3)

- **排针**：PZ254V-11-04P (1×4P, 2.54mm, 方针直插)
- **引脚连接**：
  - Pin 1：GND
  - Pin 2：RX（连接 MCU UART0 TX）
  - Pin 3：TX（连接 MCU UART0 RX）
  - Pin 4：5V（为串口屏供电）
- **串口参数**：115200bps, 8N1（SCI0 外设）
- **通信协议**：采用 Nextion 串口屏协议，指令帧以 0xFF 0xFF 0xFF 结尾

### 1.6 状态指示 LED

| LED | 颜色 | 连接引脚 | 限流电阻 | 功能 |
|-----|------|----------|----------|------|
| LED1 | 红色 (0603) | P015 (LEG_R1) | R112 (10K) | 通道 1 故障指示 |
| LED3 | 绿色 (0603) | P500 (LED_G1) | R113 (10K) | 通道 1 正常/空载指示 |
| LED2 | 红色 (0603) | P013 (LED_R2) | R110 (10K) | 通道 2 故障指示 |
| LED4 | 绿色 (0603) | P014 (LED_G2) | R111 (10K) | 通道 2 正常/空载指示 |
| LED5 | 绿色 (0603) | 5V 直供 | R114 (10K) | 电源指示（常亮） |

- LED 状态编码：故障时红灯亮绿灯灭，正常有载时绿灯常亮，正常无载时绿灯闪烁（8 周期节拍，约 400ms 亮 / 400ms 暗）

## 二、软件设计要点

### 2.1 MCU 与开发框架

- **MCU**：Renesas R7FA6E2BB (RA6E2 系列, ARM Cortex-M33, 200MHz)
- **系统时钟树**：
  - HOCO 20MHz → PLL ×10 (Div/1) → 200MHz → ICLK (CPU)
  - PCLKA = 100MHz (高速外设：GPT 定时器)
  - PCLKB = 50MHz (中速外设：I2C, SCI)
  - PCLKD = 100MHz (DAC/ADC 等)
- **开发框架**：Renesas FSP (Flexible Software Package)，BSP + HAL 层裸机架构
- **构建系统**：CMake + ARM GCC 12.2 + Ninja
- **IDE**：CLion / VS Code（CMake 集成），RASC (Renesas Smart Configurator) 图形化外设配置

### 2.2 软件架构（模块划分）

```
src/
├── hal_entry.c      ← 主入口 & 主循环（保护状态机、LED 指示、系统调度）
├── apps.c           ← 风扇控制 API（FAN_STATE_SET 继电器开关 + FAN_SPEED_SET PWM 调速）
├── bsp_uart.c       ← UART0 驱动（串口屏通信协议：发送指令、接收解析、屏幕更新）

└── hal_warmstart.c  ← BSP 启动回调（复位后引脚配置）
```

### 2.3 INA226 驱动详解 (ina226.c)

#### 初始化流程

1. **软件复位**：写配置寄存器 (0x00) = 0x8000，触发芯片内部复位，等待 10ms
2. **配置参数**：写 0x4127，含义：
   - Bit[15:9] = 0100000 → 连续测量模式
   - Bit[8:6] = 100 → 分流电压 128 次平均采样
   - Bit[5:3] = 100 → 总线电压 128 次平均采样
   - Bit[2:0] = 011 → 分流电压转换时间 8.244ms
   - 总转换时间 = (128 + 128) × 8.244ms ≈ 2.1s（128 次平均，单次 8.244ms，连续测量时自动循环）

#### 寄存器读写

- **写寄存器**：`[从机地址+W] [寄存器地址] [数据高8位] [数据低8位]`，中断等待 TX 完成
- **读寄存器**：`[从机地址+W] [寄存器地址] + Restart + [从机地址+R] [数据高8位] [数据低8位]`
- **超时保护**：每次 I2C 操作带 `I2C_TIMEOUT_MAX (0xFFFFF)` 超时计数器，防止总线死锁

#### 电流换算优化（核心亮点）

INA226 提供两种获取电流的方式：

| 方式 | 寄存器 | 需要的前置操作 | I2C 通信次数 |
|------|--------|--------------|-------------|
| 标准方式 | 电流寄存器 (0x04) | 必须先写校准寄存器 (0x05) | 初始化 1 次 + 每次读取 1 次 |
| **本项目方式** | 分流电压寄存器 (0x01) | 无需任何校准寄存器 | 每次读取 1 次 |

**换算公式推导：**
- INA226 分流电压寄存器 LSB = 2.5uV（固定值，无需校准）
- 采样电阻 R = 200mΩ (0.2Ω)
- 真实电流 I = V_shunt / R = (signed_raw × 2.5uV) / 0.2Ω = signed_raw × 12.5uA
- 但代码中直接用 `signed_raw × 0.000025f` = signed_raw × 25uA（注意：这里的 R 是 100mΩ 时的系数，代码注释说采样电阻是 100mΩ，但网表显示是 200mΩ — 这可能是代码与硬件的差异点）

**优势**：省去校准寄存器配置步骤，驱动更简洁，适用于保护阈值判断场景（不要求绝对精度，只需相对变化）。

### 2.4 主循环逻辑详解 (hal_entry.c)

#### 启动阶段

```
1. IIC_B_MASTER_Open()        → 初始化 I2C 硬件（100kHz, 中断模式）
2. INA226_Init(0x40)           → 复位+配置 U50（通道 2）
3. INA226_Init(0x41)           → 复位+配置 U49（通道 1）
4. GPT_Open/Start(timer0)      → 启动 PWM1（通道 1 风扇，P104 输出）
5. GPT_Open/Start(timer1)      → 启动 PWM2（通道 2 风扇，P112 输出）
6. UART0_Init()                → 初始化 SCI0 (115200 8N1)
7. UART0_SendCmd("page 0")    → 切换串口屏到首页
8. FAN_STATE_SET(1/2, 1)       → 两路继电器闭合
9. FAN_SPEED_SET(1/2, 25%)     → 25% 风速运行 1 秒（自检）
10. FAN_STATE_SET(1/2, 0)      → 关断继电器 1 秒（自检断开）
11. FAN_STATE_SET(1/2, 1)      → 重新闭合继电器，进入正常工作
12. 初始风速：通道 1 = 0%，通道 2 = 100%
```

#### 保护状态机（每通道独立）

```
          ┌────────────────────────────────────────────────────┐
          │                                                    │
          ▼                                                    │
    ┌──────────┐    电压<9V / >13V / 电流>90mA     ┌──────────┤
    │   fine   │ ────────────────────────────────→ │  short   │
    │ (正常)   │                                   │ (故障)   │
    └──────────┘ ←──────────────────────────────── └──────────┘
          ▲      恢复计数 ≥ 30 (1.5s) 后自动恢复         │
          │      + 启动忽略窗口 10 周期 (500ms)           │
          │                                                │
          └────────────────────────────────────────────────┘
```

- **fine 状态**：正常监测电压/电流，满足任一条件即触发保护：
  - `volt < 9.0V`：欠压（风扇断线或电源异常）
  - `volt > 13.0V`：过压（电源异常或感应电压尖峰）
  - `curr > 0.09A (90mA)`：过流/短路
  - 注意：启动忽略窗口 (startup_ignore) 内不检测，避免浪涌电流误触发
- **short 状态**：继电器断开（物理切断负载），每周期 recovery_timer 自增，达到 30 (1.5 秒) 后尝试恢复
- **恢复后**：重新闭合继电器，设置 startup_ignore = 10 (500ms) 忽略窗口

#### 串口屏交互

- **下行（MCU → 屏幕）**：每次循环更新电压/电流显示文本
  - `t1.txt="xx.xx"` → 通道 1 电压（精确到 0.01V）
  - `t3.txt="x.xxx"` → 通道 1 电流（精确到 mA）
  - `t5.txt="x.xxx"` → 通道 2 电流（精确到 mA）
  - 故障时显示 `ERR` 而非数值
- **上行（屏幕 → MCU）**：中断回调中接收
  - `65 01 05 01 FF FF FF`：页面 1 按钮 5 触发（置位 fan_update_request）
  - `71 XX 00 00 00 FF FF FF`：get 响应，XX = 0~100 占空比值
- **风速更新流程**（主循环中处理 fan_update_request 标志）：
  1. `get h0.val` → 等待 5ms → 解析通道 1 占空比
  2. `get h1.val` → 等待 5ms → 解析通道 2 占空比
  3. `get c0.val` / `get c1.val` → 读取极性复选框（当前未使用）
  4. `FAN_SPEED_SET(1/2, duty)` → 应用新占空比

#### LED 指示逻辑

- **通道故障**：红灯亮 (P015/P013 = LOW)，绿灯灭 (P500/P014 = HIGH)
- **正常有载** (curr ≥ 20mA)：红灯灭 (HIGH)，绿灯常亮 (LOW)
- **正常无载** (curr < 20mA)：红灯灭 (HIGH)，绿灯闪烁（blink_counter bit3 控制，约 400ms 周期）

### 2.5 风扇控制详解 (apps.c)

#### 继电器控制：FAN_STATE_SET(fanNAME, fanSTATE)

- `fanNAME=1`：通道 1，控制 P302 (RELAY1)
- `fanNAME=2`：通道 2，控制 P301 (RELAY2)
- `fanSTATE=1`：输出 HIGH → 三极管导通 → 继电器吸合 → 触点闭合
- `fanSTATE=0`：输出 LOW → 三极管截止 → 继电器释放 → 触点断开

#### PWM 调速：FAN_SPEED_SET(fanNAME, fanSPEED)

- 输入范围：0~100（百分比），超过 100 自动钳位
- **逻辑反转**：`actual_output_percent = 100 - fanSPEED`
  - 原因：GPT 输出的 PWM 占空比与实际风扇控制逻辑相反（可能是继电器常闭触点或风扇控制信号极性导致）
  - fanSPEED=0 → PWM=100%（全开），fanSPEED=100 → PWM=0%（全关）
- **占空比计算**：`duty_cycle_counts = (period_counts × actual_output_percent) / 100`
  - period_counts = 0xFA0 = 4000（对应 200MHz / 4000 = 50kHz，或 PCLKA/2=100MHz / 4000 = 25kHz）
  - 例：25% 风速 → actual=75% → duty=3000 counts
- **安全钳位**：`duty_cycle_counts` 不允许等于 `period_counts`（确保始终有最小脉宽）

## 三、设计亮点

### 亮点 1：双层电源架构兼顾效率与精度

DC-DC Buck (TPS62933, 效率 >95%) 负责大压差降压（12~24V → 5V），减少发热；LDO (TPS7A8101, PSRR 80dB) 负责精密稳压（5V → 3.3V），输出噪声仅 23.5uVrms。两级串联后，5V 电源轨上的开关纹波经 LDO 衰减 10000 倍后到达 3.3V 电源轨，为 INA226 和 MCU 提供极纯净的供电环境。这种"效率 + 精度"的折中方案比纯 LDO（效率低、发热大）或纯 DC-DC（纹波大、精度差）更优。

### 亮点 2：INA226 软件电流换算优化

不写校准寄存器、不读电流寄存器，仅读分流电压寄存器 (0x01) 后由软件换算电流。每次电流采集仅需 1 次 I2C 读操作（3 字节），比标准方式（先写校准寄存器 + 再读电流寄存器 = 2 次 I2C 通信）减少一半通信量。在 50ms 主循环周期内，两通道共节省 2 次 I2C 通信，为串口屏更新留出更多时间裕量。

### 亮点 3：自动恢复型通道保护（"打嗝式"保护）

采用有限状态机实现短路/过流保护，不同于传统的"熔断器烧毁"或"锁死关断"方案：检测到故障后立即关断继电器（物理隔离），1.5 秒后自动尝试恢复，若故障消失则正常工作，若故障持续则再次关断，循环往复。配合启动忽略窗口（500ms），避免恢复瞬间浪涌电流（如风扇启动电流）误触发保护。这种设计大幅提高系统可用性，特别适合负载可能临时短路的场景。

### 亮点 4：三色 LED 状态编码

每通道仅需 2 颗 LED（红+绿）+ 2 颗限流电阻，通过颜色组合 + 闪烁编码实现三种系统状态的直观指示：故障（红灯亮）、正常有载（绿灯常亮）、正常无载（绿灯闪烁）。无需任何额外显示设备，现场维护人员通过 LED 颜色即可快速判断系统状态。

### 亮点 5：继电器自检启动流程

上电后立即进行"开→25%风速→关→再开"的自检流程，主动验证继电器驱动电路和 PWM 信号通路是否正常工作。若自检阶段发现异常（如继电器不吸合），可立即发现并提示，避免在实际工作时才暴露故障。

### 亮点 6：串口屏异步交互设计

MCU 不主动轮询串口屏，而是被动等待按钮回传中断（`fan_update_request` 标志），主循环检测到标志后才发送读取指令。每条 get 指令仅等待 5ms（远小于主循环 50ms 周期），采用"请求-延时-检查响应"的非阻塞模式，避免 UART 通信阻塞主循环的保护监测。屏幕更新（电压/电流显示）也在主循环中统一处理，保证显示与保护逻辑同步。

### 亮点 7：PCB 采样电阻开尔文接法

电流采样电阻采用开尔文四线接法，将大电流通路与毫伏级电压检测走线完全分离。检测走线从采样电阻焊盘内侧引出，避免走线电阻引入测量误差。在 90mA 保护阈值下，不开尔文接法的走线压降误差可达采样电阻本身压降的 50%，开尔文接法将此误差降至可忽略水平，确保保护阈值的准确性。
